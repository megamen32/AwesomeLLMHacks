# OmniRoute

> [github.com/diegosouzapw/OmniRoute](https://github.com/diegosouzapw/OmniRoute) — единый AI-gateway на 237 провайдеров (90+ бесплатных), RTK-компрессия токенов, auto-fallback, MCP/A2A.

## Что это

Self-hosted (или hosted) OpenAI-совместимый endpoint, который прячет за одним URL зоопарк провайдеров и подписок. Подключаешь Claude Code, Codex, Cursor, Cline, Copilot или Antigravity — и они начинают видеть «один роутер», а OmniRoute сам решает, через какого провайдера гнать запрос, в какой модели и с каким fallback.

## Зачем

- Держать ключи и подписки централизованно, а не размазывать по десяткам CLI-конфигов.
- Переключать модели без переписывания каждого клиента.
- Строить профили `auto/best-coding`, `auto/best-coding-fast` и подобные.
- Выбирать модель по цене / лимитам / качеству / доступности / задаче.
- Не зависеть от одного провайдера, когда он лежит, режет лимиты или меняет правила.

## Ключевые фичи

- **237 провайдеров, 90+ бесплатных** — покрывает платные API, бесплатные тиры, локальные модели, свои прокси.
- **~1.6B free tokens / month** агрегировано из документированных free tiers, до ~2.1B в первый месяц с signup-credits. Pool-deduped — счётчик не раздувается двойным учётом общих пулов.
- **RTK + Caveman compression** — ужимает токены на 15–95% автоматически.
- **17 routing-стратегий** + **auto-fallback** — если основной провайдер не ответил, запрос уходит запасному.
- **MCP / A2A** — поддержка модель-контекст-протокола и agent-to-agent из коробки.
- **Multimodal API** — текст, изображения, etc.
- **Desktop / PWA** клиент помимо npm-пакета и Docker-образа.

## Quick start

- npm: `npm i -g omniroute`
- Docker: `docker pull diegosouzapw/omniroute`
- Desktop / PWA — через сайт [omniroute.online](https://omniroute.online).

## Когда особенно полезно

- Несколько платных подписок одновременно — OmniRoute роутит так, чтобы выжать максимум.
- Жёсткие лимиты у одного провайдера — auto-fallback разруливает без переконфига клиента.
- Разработка локальных + облачных гибридов — добавляешь Ollama / LM Studio как ещё одного провайдера.
- Эксперименты с новыми моделями — поменять провайдера = поменять routing rule, без правок клиентского кода.

## Ограничения

- Это **шлюз**, а не замена провайдеров: лимиты и квоты провайдеров остаются их лимитами. Бесплатные тиры могут меняться.
- Free-tier aggregation считается честно (pool-deduped) — не ждите «10B токенов из воздуха», цифра в README это уточняет.
- Self-host требует своего сервера / Docker-хоста. Hosted-вариант зависит от автора проекта.
- Routing-логика не всегда очевидна — для сложных политик стоит читать `docs/reference/`.
